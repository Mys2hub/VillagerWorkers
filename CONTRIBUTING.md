# Contributing to VillagerWorkers

Thank you for considering contributing to the VillagerWorkers project! We welcome contributions of all kinds. Here's a guide to help you get started:

## Reporting Issues
If you encounter any bugs or have suggestions for improvements:
1. Search the [issue tracker](https://github.com/Mys2hub/VillagerWorkers/issues) to see if someone else has already reported your issue.
2. If not, open a new issue with a clear description of the problem or suggestion.

## Submitting Changes
1. **Fork the Repository**: Start by forking the VillagerWorkers repository to your own GitHub account.
2. **Create a Branch**: Create a new branch for your feature or bug fix.
   - You can do this with: `git checkout -b your-feature-branch`.
3. **Make Changes**: Implement your changes and test them thoroughly.
4. **Commit Your Changes**: Ensure your commit messages are clear and descriptive.
   - Use present tense: "Add feature" instead of "Added feature".
5. **Push Your Changes**: Push your changes back to your forked repository.
   - Use: `git push origin your-feature-branch`.
6. **Create a Pull Request**: Navigate to the original repository and submit a pull request from your feature branch to the main branch.
